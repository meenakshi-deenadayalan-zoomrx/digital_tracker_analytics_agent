## Directory Purposes

**Project Root:**
- Purpose: MCP server code + skill instructions
- Contains: Python service files, configuration, agent skills
- Key files: `mcp_server.py`, `config.py`, `pyproject.toml`

**`skills/`:**
- Purpose: Claude agent instruction sets (separate from MCP tools)
- Contains: SKILL.md orchestrators + reference markdown docs
- Organization: Two agent systems (DTSA for diagnostics, DTAA for analytics)

**`skills/dtsa/`:**
- Purpose: DTSA orchestrator — routes diagnostic requests to DTSA sub-skills
- Contains: `SKILL.md` (routing logic)
- Depends on: Sub-skills dtsa-database, dtsa-diagnostics, dtsa-selectors, dtsa-ticketing

**`skills/dtsa-database/`:**
- Purpose: Database query helpers for volume analysis
- Contains: `SKILL.md`, `references/volume_query_patterns.md`

**`skills/dtsa-diagnostics/`:**
- Purpose: Channel/platform reference for troubleshooting
- Contains: `SKILL.md`, `references/channel_reference.md`

**`skills/dtsa-selectors/`:**
- Purpose: CSS/XPath selector inventory + playwright testing
- Contains: `SKILL.md`, `references/selector_inventory.md`

**`skills/dtsa-ticketing/`:**
- Purpose: Phabricator task creation helpers
- Contains: `SKILL.md`, `references/remarkup_template.md`

**`skills/dtaa/`:**
- Purpose: DTAA orchestrator — routes analytics requests to channel skills
- Contains: `SKILL.md` (routing table), `references/` (schema, guidelines, SOV template)

**`skills/dtaa-<channel>/` (Exploratory Skills):**
- Purpose: Initial data exploration for a marketing channel (ads, web, emails, posts, search)
- Examples: `dtaa-ads/`, `dtaa-web/`, `dtaa-emails/`, `dtaa-posts/`, `dtaa-search/`
- Contains: `SKILL.md` (exploratory queries), `references/query_patterns.md`

**`skills/dtaa-<channel>-metrics/` (Metrics Skills):**
- Purpose: Calculated metrics (SOV, Reach, Frequency, Share of Attention)
- Examples: `dtaa-ads-metrics/`, `dtaa-web-metrics/`, `dtaa-emails-metrics/`, `dtaa-posts-metrics/`, `dtaa-search-metrics/`
- Contains: `SKILL.md` (metric calculations referencing SOV template)

**`.planning/codebase/`:**
- Purpose: Architecture + convention documentation (generated by GSD)
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, CONCERNS.md

## Key File Locations

**Entry Points:**
- `mcp_server.py`: Main MCP server — spawned by Claude Desktop via `stdio_server()` event loop (line 240)
- `config.py`: Settings object loaded on import — configures all services

**Configuration:**
- `config.py`: Pydantic BaseSettings class mapping env vars
- `.env`: Secrets file (ignored by git, see `.gitignore`)
- `.env.example`: Safe template showing required vars

**Core Logic:**
- `mysql_service.py`: Database queries (lines 19–44)
- `github_service.py`: Git commit history + diffs (lines 36–54 for dispatch, 68–211 for implementations)
- `playwright_service.py`: Selector testing (lines 23–73)
- `phabricator_service.py`: Task creation (lines 18–64)
- `filesystem_service.py`: File reading + grep (lines 52–208)

**Database:**
- `dtsa_database.py`: SQLAlchemy session factory (lines 24–44)

**Agent Skills (for Claude):**
- `skills/dtsa/SKILL.md`: DTSA routing orchestrator
- `skills/dtaa/SKILL.md`: DTAA routing orchestrator
- `skills/dtaa/references/sov_base_template.md`: Template for all metric calculations

## Naming Conventions

**Files:**
- Python service: `<domain>_service.py` (e.g., `mysql_service.py`, `github_service.py`)
- Configuration: `config.py`
- Database: `<domain>_database.py` (e.g., `dtsa_database.py`)
- Entry point: `mcp_server.py`
- Agent skill: `SKILL.md` (located in `skills/<agent>[-channel]/`)
- Reference docs: `<topic>.md` (located in `references/` subdirs)

**Functions (service methods):**
- Public tool handler: `<action>()` e.g., `execute_read_query()`, `test_selector()`, `create_task()`
- Internal helper: `_<action>()` e.g., `_resolve_safe()`, `_use_github_api()`, `_parse_iso()`

**Variables:**
- Environment settings: `env.<VAR_NAME>` from `config.env`
- Service constants: `UPPERCASE` (e.g., `FORBIDDEN`, `ALLOWED_REPOS`, `MAX_FILE_BYTES`)

**Classes:**
- Service class: `Dtsa<Domain>Service` e.g., `DtsaMysqlService`, `DtsaGithubService`
- Settings class: `Settings` (in `config.py`)

## Where to Add New Code

**New Tool (e.g., add `dtsa_twilio_sms` for SMS verification):**
1. Create `twilio_service.py` in project root
2. Define `DtsaTwilioService` class with async static method (follow `phabricator_service.py` pattern)
3. Add import in `mcp_server.py` line 29
4. Add tool definition in `list_tools()` (lines 40–205)
5. Add case handler in `call_tool()` dispatcher (lines 211–237)
6. Add credentials to `config.py` (e.g., `DTSA_TWILIO_API_KEY`)

**New Agent Skill (e.g., add `dtsa-api-endpoints` for endpoint testing):**
1. Create `skills/dtsa-api-endpoints/` directory
2. Create `SKILL.md` with prompt instructions (reference existing `dtsa-database/SKILL.md`)
3. Create `references/` subdirectory with supporting docs (e.g., `api_inventory.md`)
4. Update `skills/dtsa/SKILL.md` routing table to include new skill

**New DTAA Channel (e.g., add `dtaa-tiktok` for TikTok analytics):**
1. Create exploratory skill:
   - `skills/dtaa-tiktok/SKILL.md` (sample queries referencing TikTok API)
   - `skills/dtaa-tiktok/references/query_patterns.md`
2. Create metrics skill:
   - `skills/dtaa-tiktok-metrics/SKILL.md` (metric calculations)
   - Reference `dtaa/references/sov_base_template.md` for SOV pattern
3. Update `skills/dtaa/SKILL.md` routing table

**Utilities (shared helpers):**
- Create in project root as `<domain>_util.py`
- Import in relevant service files
- Example: If multiple services need URL encoding, create `url_util.py` with helpers
